
# DevOps: Containerization and Container Orchestration
### Graded Project on Orchestration and Scaling

An end-to-end production deployment pipeline transforming a monolithic full-stack MERN web application into a cloud-native, auto-scaling microservices architecture hosted on Amazon EKS.

## 🛠️ Complete Technical Stack

* **Application Layer:** ReactJS (Frontend), NodeJS & ExpressJS (Backend Server), MongoDB (Database)
* **Containerization:** Docker Engine (Multi-stage builds, Alpine base images optimized to 53MB)
* **Image Registry:** Amazon Elastic Container Registry (ECR)
* **CI/CD Automation:** Jenkins Automation Server (EC2 hosted, Declarative Pipelines, GitHub Webhooks)
* **Orchestration & Deployment:** Amazon Elastic Kubernetes Service (EKS), Managed Node Groups, Helm Charts (v3)
* **Production Observability:** Amazon CloudWatch (Container Insights & Fluent Bit centralized logs)
* **ChatOps Automation:** Amazon SNS (Automated Slack/Telegram deployment status alerts)

## 🚀 Key Infrastructure Milestones Executed
1. **Source Control Management:** Upstream synchronization workflows via fork mechanics.
2. **Optimized Containerization:** Reduced attack surfaces and image footprints using lean Docker layers.
3. **Continuous Integration (CI):** Fully automated push-triggered compilation, verification, and artifact registry delivery.
4. **Cloud-Native Deployment:** Declarative manifest packages managed cleanly via Helm releases.
5. **High-Scaling & Monitoring:** Node tracking, load balancing, log aggregation, and real-time operational feedback.
